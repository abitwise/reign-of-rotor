- `memory-bank/ARCHITECTURE.md` — Architecture, design rules, integration constraints  
- `memory-bank/PRODUCT.md` — Domain, business rules, intended behavior  
- `memory-bank/CONTRIBUTING.md` — Coding standards, testing rules, conventions

# WORKFLOW RULES

## Step 1: Before you start to create ticket(s)
- Read the info about existing tickets from TICKETS.md
  - If possible, map the feature(s) to already described feature(s). Ticket duplication is bad.
  - If you can't find existing matches for the requirements, proceed to step 2
  - If you find existing matches, see if the existing ticket(s) needs updating.
  - Proceed to Step 2 only with new requirements, skip previously mapped/implemented requirements.

## Step 2: Create ticket(s)
- Think through what these new requirements are about
- Focus on creating correct and readable ticket(s)
- Don't implement any code changes, this is just about documenting
- See which Phase the ticket goes under and save it in `memory-bank/plans/TICKETS.md`
  - In case it is standalone or future upgrade, put it under Backlog
